Relationship map preview

入口、项目、场景、规则和 skill，各自有清晰边界。

最小版用 JSON 描述关系，用 skill 承接读取方式。AI 先看索引，按需加载上下文，并把任务状态留给多个工具共享。

Entry

config/entry.json

定义默认读取顺序、隐私边界和安装位置，是 AI 进入仓库后的第一个小入口。

Current

runtime/current.json

记录当前任务或项目指针，让新会话可以从一个轻量状态开始恢复。

Project index

config/projects/index.json

只保存候选列表。真实项目详情可以按需放进私有项目 JSON。

Selected detail

matched project / task / rule

只有当前请求需要时，AI 才继续加载选中的项目详情、任务状态、规则说明或更长文档。这样能减少噪音，也更容易解释为什么读了某个文件。

Skill

skill/ai-context

把这套读取习惯安装到本地 AI 工具里，让不同工具共用同一个入口约定和任务指针。

01 小入口 先读 entry，知道应该怎么进入。
02 状态指针 再读 current，恢复当前任务位置。
03 关系索引 用项目索引选择候选上下文。
04 按需加载 只打开本次请求需要的资料，减少聊天窗口上下文。
