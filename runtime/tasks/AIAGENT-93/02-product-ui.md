## 已确认交互规则

- 中间生成的原型、文档、技术方案、验收报告等关键产物，需要用户确认。
- 未确认时，当前任务停留在 `Pending User Review`。
- 用户确认后才能进入下一 Gate。
- 技术方案确认前，不启动开发 Agent / Codex worker。
- 桌面端左侧区域直接放任务栏；运行 Agent、Workspace、验收与返工、配置作为右侧任务详情模块展示。
- 新建任务不放在每个任务详情里；桌面端从任务栏 `+` 打开弹窗，移动端从当前任务条打开弹窗。
- 新建任务弹窗中，任务描述改为多行大输入框，下面支持上传图片和文件；附件先进入本地任务数据，后续可替换真实上传接口。
- 左侧任务栏状态色：待启动白色；执行中蓝色，并用浅蓝色横向进度条表示流程进度；待确认浅黄色；已完成浅绿色。
- 新创建且尚未开始的任务固定为 `G0 / Draft`，展示为“待启动”；用户在 PM 聊天里说“开始”“启动项目”等语言后，PM 才推进到启动会/边界探索。
- 右侧展示“任务流程”而不是固定全量 Gate：未启动时只显示 `G0 待启动` 和“流程待 PM 启动会后生成”；启动会后显示 PM 为当前任务生成的专属流程。小 BUG / 中型需求 / 跨项目大型需求的流程可以不同。
- 控制台主操作改成 PM Chat First：中间是 PM 聊天窗口，支持文字、图片和文件；需求、纠错、验收反馈都从这里发给 PM。
- 右侧只作为状态面板展示任务上下文、Gate、产物列表、Agent / Worker、验收返工记录，不再单独展示 PM 决策过程。
- 任务上下文中的“涉及项目”只显示 PM/Discovery 判断出的业务项目；还没开始判断时显示待判断空态，不显示控制台自身代码路径。
- 产物列表展示所有已生成的文档、原型、设计、方案和验收报告；可切换过滤 `PendingUserReview` 待确认产物；每个产物可打开，也可一键把产物路径或链接带到 PM 输入框；没有任何产物时不显示该面板；没有验收返工时，不显示验收返工面板。
- 未开始任务只显示 PM，不提前显示 mock worker 或开发员工。
- 右侧 Agent / Worker 可以选择某个员工查看输出；选中非 PM 员工时，中间区域切成只读输出视图，不提供直接聊天输入，避免绕过 PM 调度。
- PM 的分析、判断和下一步动作直接作为聊天回复展示；用户消息靠右，PM 回复靠左。
- 左侧任务栏、中间 PM 聊天、右侧状态面板各自独立滚动，互不影响。
